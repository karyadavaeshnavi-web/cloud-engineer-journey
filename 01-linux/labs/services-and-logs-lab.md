# Linux Services and Logs Lab

## Objective

Learn how Linux services are managed and how logs are used for troubleshooting.

---

## Environment

Ubuntu Linux

---

## Commands Executed

systemctl status cron

ls /var/log

journalctl -n 20

journalctl -u cron

---

## Service Inspection

Inspected the cron service using:

systemctl status cron

Observed:

- Service was active and running
- Service was enabled
- Main PID was displayed
- Memory usage was displayed
- CPU usage was displayed
- Recent service logs were available

---

## Log Exploration

Explored the Linux log directory:

/var/log

Observed files:

- auth.log
- syslog
- kern.log
- dpkg.log

These files contain authentication, system, kernel, and package management logs.

---

## journalctl Usage

Used journalctl to view recent logs.

Commands:

journalctl -n 20

journalctl -u cron

Observed service activity and recent log entries generated by the cron service.

---

## Key Learnings

- Services run continuously in the background.
- systemd manages Linux services.
- systemctl controls services.
- Logs are essential for troubleshooting.
- journalctl provides access to service and system logs.
- Linux stores logs inside /var/log.

---

## Outcome

Successfully inspected services, explored system logs, and learned how Linux records and manages service activity.
